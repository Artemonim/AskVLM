## Основной flow

1. Запуск транскрибатора
1.1. Вход
1.1.1. файл (аудио/видео)
1.1.2. папка (плоская); рекурсивный режим — в планах
1.2. Выход (директория назначения и формат экспорта)
1.2.1. txt | srt | vtt | json (docx/odt/md — в планах)

2. Настройки обработки (до старта)
2.1. Движок: Авто (faster-whisper; выравнивание/слова — через whisperx при наличии)
2.2. Язык: авто или заданный код (например, ru)
2.3. Диаризация: вкл/выкл (VAD+спикеры; улучшения — в планах)
2.4. Диалог‑блоки (LLM форматирование): вкл/выкл
2.5. Модель: Авто (подбор по VRAM/скорости); ручной выбор — в «Расширенных» (опционально)
2.6. Устройство/тип вычислений: авто (GPU → FP16, иначе INT8/FP32)
2.7. Выбор движка и модели скрыт в UI: по умолчанию «Авто»; переключатели появляются лишь в «Расширенных».

3. Автоматическая обработка
3.1. Сплошной текст — без постобработки
3.2. Фразы — стандартный вывод Whisper: «тайминг → фраза»
3.3. Диалог — текст разбит на блоки по говорящим; при включении LLM — повышение читабельности
3.4. Выравнивание слов (whisperx) — опционально, при доступности

4. Ручная обработка
4.1. WYSIWYG‑редактор: просмотр/правка текста
4.2. В планах: сплит/мердж сегментов, смена спикера, удаление таймкодов, поиск/замена, спеллчек

5. Экспорт
5.1. TXT — полный текст
5.2. SRT/VTT — подстрочники с таймкодами и спикерами
5.3. JSON — сегменты со структурой {speaker, start, end, text}

6. CLI (быстрый сценарий)
6.1. Пример: `python cli.py transcribe -i INPUT -o output --engine whisperx --diarization --dialog-blocks --export srt`

7. Статус и прогресс
7.1. GUI — статус‑бар, не блокирующие операции
7.2. CLI — информативный вывод; ETA — в планах

8. Секреты и модели
8.1. HF_TOKEN — для моделей диаризации (ENV)
8.2. LLM_GGUF_PATH — путь к локальной модели для форматирования (ENV)

11. Решение по выбору движка и модели
11.1. В UI не показываем переключатель «Whisper/WhisperX». По умолчанию используем faster-whisper; если установлен whisperx — включаем выравнивание/слова при выборе опции «Выравнивание».
11.2. Пресеты моделей скрыты. Режим «Авто» подбирает модель по VRAM/скорости. Для продвинутых пользователей доступен ручной выбор в «Расширенных».

9. Ошибки и устойчивость
9.1. OOM/GPU недоступен — авто‑даунгрейд типа вычислений; рекомендация отключить диаризацию/LLM
9.2. Неподдерживаемый формат — авто‑конверт в WAV 16 kHz mono

10. UX‑принципы
10.1. Разумные дефолты и запоминание последних настроек
10.2. Ясные сообщения об ошибках и возможные шаги исправления
